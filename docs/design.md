# 卖鱼记账小程序设计文档

## 概述

一款微信小程序，用于卖鱼生意的销售记账和收款管理。支持多店铺独立运营，店主邀请家庭成员协作。核心流程：记录销售金额 → 顾客付款后标记已收 → 汇总统计已收收入。

## 技术方案

- **平台**：微信小程序（原生开发）
- **前端**：WXML + WXSS + JavaScript
- **后端**：微信云开发（云数据库 + 云函数）
- **存储**：微信云数据库
- **登录**：微信登录（获取 openid）

## 数据模型

### shops 集合（店铺）

```json
{
  "_id": "自动生成",
  "name": "老王鱼摊",
  "inviteCode": "382916",
  "ownerId": "微信openid",
  "createTime": 1684659000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 店铺名称 |
| inviteCode | String | 6位邀请码，用于成员加入 |
| ownerId | String | 创建者的微信 openid |
| createTime | Number | 创建时间戳 |

### members 集合（成员关系）

```json
{
  "_id": "自动生成",
  "shopId": "店铺id",
  "userId": "微信openid",
  "nickName": "微信昵称",
  "role": "owner",
  "joinTime": 1684659000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| shopId | String | 所属店铺 id |
| userId | String | 微信 openid |
| nickName | String | 微信昵称 |
| role | String | owner=店主，member=成员 |
| joinTime | Number | 加入时间戳 |

### sales 集合（销售记录）

```json
{
  "_id": "自动生成",
  "shopId": "店铺id",
  "amount": 120.50,
  "note": "张三买了10斤草鱼",
  "date": "2026-05-21",
  "time": "14:30",
  "createTime": 1684659000000,
  "status": "pending"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| shopId | String | 所属店铺 id |
| amount | Number | 销售金额（元） |
| note | String | 备注（可选） |
| date | String | 日期，格式 YYYY-MM-DD |
| time | String | 时间，格式 HH:mm，自动获取 |
| createTime | Number | 时间戳，用于排序 |
| status | String | pending=未收款，collected=已收款 |

## 页面设计

### 1. 启动页（首次使用）

- 微信自动登录
- 两个选项：创建店铺 / 加入店铺
- 已有店铺的用户自动进入上次使用的店铺

### 2. 创建店铺页

- 输入店铺名称
- 点击"创建"后生成 6 位邀请码
- 显示邀请码 + 二维码（可截图分享给家人）
- "进入店铺"按钮

### 3. 加入店铺页

- 输入 6 位邀请码
- 点击"加入"
- 加入成功后进入该店铺

### 4. 首页（待收款）

- 顶部：今日日期 + 待收总额
- 输入区域：金额输入框 + 备注输入框 + "记录"按钮
- 列表：所有未收款记录（status=pending），按时间倒序
- 左滑操作："已收"按钮，点击后将 status 改为 collected
- 记录样式：
  ```
  ¥120.50          14:30
  张三买了10斤草鱼
  ```

### 5. 汇总页

- 默认显示：今日已收总额（大字）+ 笔数
- 下方展示今天的已收记录列表
- 切换视图：
  - 日汇总：按天列表，每天的已收金额和笔数
  - 月汇总：按月列表，每月的已收金额和笔数
  - 日期筛选：选择起止日期，查看指定区间汇总
- 日汇总列表样式：
  ```
  05月21日（今天）  ¥580.00  5笔
  05月20日          ¥420.00  3笔
  ```
- 点击某天进入详情，查看该天所有已收记录

### 6. 记录详情页

- 显示某一天的所有已收记录列表
- 每条显示：金额、备注、收款时间

### 7. 店铺设置页

- 显示店铺名称和邀请码（可复制）
- 显示已加入的成员列表（头像 + 昵称 + 角色）
- 店主可转让店主给其他成员（每个成员旁边有"转让店主"按钮）
- 其他操作区域：
  - 我的店铺（进入店铺列表页）
  - 生成二维码（生成小程序码供扫码加入）
  - 退出店铺（普通成员可退出，店主需先转让）

### 8. 我的店铺页

- 显示用户加入的所有店铺列表（店铺图标 + 名称 + 角色 + 人数）
- 当前使用的店铺高亮显示，带"当前"标签
- 点击其他店铺可切换
- 底部两个操作按钮：创建店铺 / 加入店铺
- 说明文字：每个店铺数据完全独立，互不影响

### 9. 转让店主页

- 顶部警告提示：转让后变为普通成员，无法撤销
- 选择新店主：列出所有普通成员（头像 + 昵称）
- 单选，点击选中，显示绿色勾
- "确认转让"按钮（红色）

## 导航结构

- 底部 Tab 栏三个入口：记账、汇总、设置
- 记账 → 首页（待收款列表）
- 汇总 → 汇总统计页
- 设置 → 店铺设置页（包含我的店铺、退出店铺等功能入口）

## 多店铺协作

- 每个店铺数据完全隔离，通过 shopId 区分
- 店主创建店铺后生成 6 位邀请码 + 小程序码
- 家庭成员输入邀请码或扫码加入店铺
- 同一店铺的所有成员共享数据，都可记账和收款
- 支持一个用户加入多个店铺（在"我的店铺"页面切换）
- 店主只能转让不能解散，转让后变为普通成员
- 普通成员可自由退出店铺
- 所有操作通过微信登录验证身份

## 核心流程

1. 用户打开小程序 → 微信登录
2. 首次使用：创建店铺或输入邀请码加入
3. 已有店铺的用户自动进入上次使用的店铺
4. 进入店铺首页 → 输入金额和备注 → 点击"记录"→ 自动获取当前日期时间 → 保存为 pending
5. 顾客付款 → 左滑点击"已收"→ 状态改为 collected
6. 汇总页统计所有 status=collected 的记录

## 店铺管理流程

- **创建店铺**：输入店铺名 → 生成邀请码 + 小程序码 → 分享给家人
- **加入店铺**：输入邀请码或扫码 → 自动加入 → 成为普通成员
- **切换店铺**：设置 → 我的店铺 → 点击其他店铺切换
- **转让店主**：设置 → 成员列表 → 点击"转让店主"→ 选择新店主 → 确认转让
- **退出店铺**：设置 → 其他操作 → 退出店铺（店主需先转让）
